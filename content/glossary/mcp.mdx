## What is MCP?

The **Model Context Protocol (MCP)** is an open standard introduced by Anthropic in November 2024. It defines how AI applications connect to external tools, data sources, and services through a unified interface.

Think of it as **USB-C for AI**: one protocol, many connectors.

## Architecture

```
Host (IDE/Agent) ↔ MCP Client ↔ MCP Server ↔ Tools/Data
```

### Components

- **Host**: The AI application (Cursor, Claude Desktop, VS Code)
- **Client**: Maintains 1:1 connections with servers
- **Server**: Exposes tools, resources, and prompts
- **Tools**: Callable functions (search, create, deploy)
- **Resources**: Readable data (files, database records)
- **Prompts**: Reusable prompt templates

## Why MCP matters

Before MCP, every AI tool built custom integrations. MCP standardizes this:

1. **Write once, connect everywhere** — one MCP server works with any MCP client
2. **Composability** — combine multiple servers (GitHub + Linear + Postgres)
3. **Security** — explicit permission model for tool access
4. **Ecosystem** — thousands of community servers on [mcp.so](https://mcp.so)

## Adoption timeline

- **Nov 2024**: Anthropic announces MCP
- **Mar 2025**: OpenAI adopts MCP across products
- **Dec 2025**: AAIF/Linux Foundation governance

## MCP vs APIs

MCP is not a replacement for REST APIs. It's a **discovery and invocation layer** designed specifically for LLM agents — with schema descriptions optimized for model understanding.
