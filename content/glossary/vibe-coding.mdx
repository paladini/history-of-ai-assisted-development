## Origin

On February 2, 2025, Andrej Karpathy tweeted:

> "There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists."

The term quickly spread across developer communities, tech press, and conference talks.

## What it means

**Vibe coding** describes a development style where:

- You accept AI-generated code with minimal line-by-line review
- Focus shifts from implementation details to product feel and outcomes
- "The code even exists" becomes secondary to whether the feature works
- Iteration happens through natural language, not manual editing

## The debate

### Advocates
- AI code quality has improved enough for rapid prototyping
- Developer time is better spent on architecture and UX than syntax
- Speed of iteration matters more than understanding every line

### Critics
- Unreviewed code accumulates technical debt
- Security vulnerabilities hide in generated code
- Developers lose debugging skills and codebase understanding
- "Vibe" is not a substitute for engineering rigor

## Connection to the trust paradox

Stack Overflow 2025 data shows **84% adoption but only 29% trust** — vibe coding is arguably a symptom of this gap. Developers use AI extensively but don't fully trust it, leading to a middle ground of loose acceptance.
