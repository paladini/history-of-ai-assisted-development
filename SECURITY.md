# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

This is a static documentation site with no server-side runtime, user authentication, or data collection. Security concerns are limited to:

- **Dependency vulnerabilities** — report via GitHub Security Advisories or open an issue
- **XSS via content** — if you find a way to inject scripts through timeline YAML or glossary MDX, please report privately

To report a security issue, open a [GitHub Security Advisory](https://github.com/paladini/history-of-ai-assisted-development/security/advisories/new) or email the repository owner.

Please do not open public issues for security vulnerabilities.

## Scope

Out of scope:
- Social engineering against maintainers
- Denial of service against GitHub Pages hosting
- Issues in third-party services linked from the timeline (report to those services directly)
